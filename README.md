# 3D 仓库货位巡检看板

基于 Three.js + React 的三维可视化仓库货位巡检管理工具，支持库存快照展示、冲突检测与人工确认、历史盘点记录回放等功能。

## 功能特性

- **3D 可视化场景**：货架、货位、托盘的沉浸式三维展示，支持鼠标旋转、缩放、平移
- **布局配置导入**：支持 JSON 格式的仓库布局配置文件导入
- **多维度筛选**：按货架、货位状态筛选显示
- **点击定位**：点击货位高亮显示，同步查看详细信息和托盘列表
- **状态颜色编码**：不同状态对应不同颜色，直观展示库存状态
- **自动冲突检测**：自动检测同一货位多托盘、重复托盘号、未知货位等异常
- **人工确认处理**：对检测出的冲突进行确认处理，记录处理状态
- **历史回放**：支持盘点记录时间轴回放，查看库存变化历程
- **异常清单导出**：将检测到的异常导出为 CSV 格式文件
- **刷新恢复**：浏览器刷新后保留筛选、确认状态、回放记录、导出数量和视角
- **视角复位**：一键重置所有状态和相机视角

| 2 | 货位多托盘 | 货位 **C-03-1** 被 2 个托盘占用：PLT-2024-0023、PLT-2024-0012 |

| 4 | 未知货位 | 托盘 **PLT-2024-U01** 位于未知货位 `UNKNOWN-SLOT-999` |

## 主操作流程

### 1. 载入样例仓库

启动应用后，系统会自动载入样例仓库数据，3D 场景会渲染出 3 个货架、45 个货位和 35 个托盘。

### 2. 定位冲突货位

1. 在右侧 **异常清单** 面板中，找到类型为「货位多托盘」的冲突记录
2. 点击「定位货位」按钮，3D 场景会自动选中该货位并高亮显示
3. 左侧 **货位详情** 面板会显示该货位的详细信息，包括：
   - 货位编号（如 B-02-1）
   - 所属货架
   - 位置信息（列、层）
   - 托盘列表（会显示 2 个托盘）
   - 冲突预警信息

### 3. 人工确认处理

1. 在右侧异常清单中，找到该冲突记录
2. 点击「人工确认」按钮
3. 冲突记录会标记为「已确认」，并记录确认时间
4. 右上角待处理计数会减少

### 4. 导出异常清单

1. 点击右侧异常清单面板右上角的「导出」按钮
2. 系统会自动下载 CSV 格式的异常清单文件
3. 文件名格式：`异常清单_YYYY-MM-DD.csv`
4. 导出包含以下字段：序号、冲突类型、描述、相关ID、是否已确认、确认时间

## 错误提示机制

系统对以下异常情况会显示可见的失败提示：

| 异常类型 | 提示方式 | 触发条件 |
|----------|----------|----------|
| 重复托盘号 | 右侧异常清单 + Toast 通知 | 同一托盘号出现在多个位置 |
| 未知货位 | 右侧异常清单 + Toast 通知 | 托盘的货位ID不存在于货位列表中 |
| 损坏布局配置 | 右侧异常清单 + Toast 通知 | 导入JSON解析失败或数据结构不完整 |
| 空数据集 | 右侧异常清单 + Toast 通知 | 货架或货位数量为0 |

所有提示都会在页面右上角以 Toast 形式显示 4 秒后自动消失。

## 刷新恢复机制

浏览器刷新后，以下状态会自动恢复（数据保存在 LocalStorage）：

- ✅ **筛选条件**：货架筛选、状态筛选
- ✅ **确认状态**：已确认的冲突记录
- ✅ **回放记录**：当前播放的历史盘点记录索引
- ✅ **导出数量**：已导出异常清单的次数
- ✅ **选中货位**：当前选中的货位ID
- ✅ **相机视角**：当前相机位置和观察目标

点击工具栏的「复位」按钮可以清除所有保存的状态并重置相机视角。

## 布局配置导入

支持导入自定义 JSON 格式的仓库布局配置。

### JSON 格式示例

```json
{
  "version": "1.0.0",
  "name": "我的仓库",
  "shelves": [
    {
      "id": "shelf-1",
      "name": "1号货架",
      "position": { "x": 0, "z": 0 },
      "rows": 1,
      "columns": 5,
      "levels": 3
    }
  ],
  "slots": [
    {
      "id": "01-01-1",
      "shelfId": "shelf-1",
      "row": 0,
      "column": 0,
      "level": 0,
      "status": "occupied"
    }
  ],
  "pallets": [
    {
      "id": "p-001",
      "palletNo": "PLT-0001",
      "slotId": "01-01-1",
      "status": "normal",
      "sku": "SKU-001",
      "quantity": 100,
      "lastCheckTime": "2024-01-15 09:30:00"
    }
  ],
  "inventoryRecords": [
    {
      "id": "rec-001",
      "timestamp": "2024-01-15 08:00:00",
      "note": "期初盘点",
      "pallets": []
    }
  ]
}
```

### 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| version | string | 布局配置版本号 |
| name | string | 仓库名称 |
| shelves | array | 货架列表 |
| shelves[].position | object | 货架在3D场景中的位置坐标 |
| shelves[].columns | number | 货架列数 |
| shelves[].levels | number | 货架层数 |
| slots | array | 货位列表 |
| slots[].status | enum | `empty` \| `occupied` \| `conflict` \| `warning` |
| pallets | array | 托盘列表 |
| pallets[].status | enum | `normal` \| `damaged` \| `expired` \| `unknown` |
| inventoryRecords | array | 历史盘点记录 |
